This repository acts as an *opinionated* template for Python projects, providing a standardized structure and configuration to streamline development and ensure best practices.

Core technology choices:

- [uv](https://docs.astral.sh/uv/) for project management and packaging
- [ruff](https://docs.astral.sh/ruff/) for linting and formatting
- [DVC](https://dvc.org/) for data version control
- [pre-commit](https://pre-commit.com/) for managing git hooks
- [pytest](https://docs.pytest.org/en/stable/index.html) and [coverage](https://coverage.readthedocs.io/en/stable/) for testing and code coverage
- [MkDocs](https://www.mkdocs.org/) for documentation
- [poethepoet](https://poethepoet.natn.io/) for task management
- [EditorConfig](https://editorconfig.org/) for maintaining consistent coding styles

## Usage

Just clone this repository and use it as a starting base. If you use Github, you can also use Github templating feature.
